## 1. 系统概述

skill-guard 是一个用 Go 语言编写的命令行安全扫描工具，实现对 AI 技能（Skill）文件的自动化安全检测。

### 1.1 设计目标

- **零依赖部署：** 编译为单一可执行文件，无外部运行时依赖
- **高性能：** 利用 Go 并发模型，实现毫秒级批量文件扫描
- **可扩展：** 规则引擎采用接口化设计，支持自定义规则
- **CI/CD 友好：** 语义化退出码、结构化 JSON 输出

### 1.2 技术栈

| 技术 | 用途 |
|------|------|
| Go 1.22+ | 开发语言 |
| Go `flag` 或 `cobra` | 命令行参数解析 |
| Go `regexp` | 正则匹配引擎（RE2） |
| Go `os/filepath` | 目录遍历与文件操作 |
| Go `sync` | 并发控制 |
| Go `encoding/json` | JSON 输出 |
| Go `encoding/yaml` | 配置文件与规则文件加载 |

## 3. 核心数据流

### 3.1 主扫描流程

```
用户输入
    │
    ▼
┌──────────────┐     ┌──────────────────┐
│   CLI 参数解析 │────▶   配置加载与合并   │
└──────────────┘     └────────┬─────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   目录遍历发现文件  │
                    │   应用 ignore 规则  │
                    │   扩展名过滤        │
                    └────────┬─────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   文件预检查        │
                    │   编码检测 · 大小检查 │
                    └────────┬─────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   Goroutine 池分发  │
                    │   并发扫描文件       │
                    └────────┬─────────┘
                              │
                    ┌─────────▼─────────┐
                    │    规则引擎匹配      │
                    │  关键字预过滤 → 正则  │
                    └─────────┬─────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   结果聚合 · 排序    │
                    │   严重级别过滤       │
                    └────────┬─────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   输出渲染 · 退出码  │
                    │  终端 / JSON / Quiet│
                    └──────────────────┘
```

### 3.2 流程说明

1. **启动阶段：** 解析 CLI 参数 → 加载配置文件 → 合并为统一 Config
2. **文件发现阶段：** 递归遍历目标目录 → 应用 ignore 规则 → 过滤扩展名 → 构建扫描文件列表
3. **预检阶段：** 检查文件编码（非 UTF-8 跳过）、文件大小（超限跳过）
4. **并发扫描阶段：** 通过 channel 分发文件到 goroutine 池 → 每文件独立扫描
5. **规则匹配阶段：** 关键字预过滤 → 正则匹配 → 记录匹配结果
6. **聚合阶段：** 从 channel 收集结果 → 去重 → 按严重级别排序 → 应用 severity 过滤
7. **输出阶段：** 根据输出模式（终端/JSON/Quiet）渲染结果 → 设置退出码

## 4. 技术决策

### 4.1 并发模型

**方案： Worker Pool 模式**

```
文件列表 ──► 任务 Channel ──► [Worker 1] [Worker 2] ... [Worker N]
                                    │
                                    ▼
                             结果 Channel ──► 聚合
```

- Worker 数量默认为 `runtime.NumCPU()`
- 可通过 `--concurrency` 参数调整
- 生产者（文件发现）与消费者（文件扫描）解耦
- 使用 `sync.WaitGroup` 等待所有 Worker 完成

### 4.2 规则匹配策略

**两级过滤：**

```
文件内容 → 关键字预过滤（快速字符串查找）
              │ 命中
              ▼
          正则匹配（RE2，O(n) 保证）
              │ 命中
              ▼
          记录 MatchResult
```

- 关键字预过滤使用 `strings.Contains`，避免对无关联的文件执行昂贵的正则匹配
- 正则使用 Go `regexp` 包（RE2 语法），保证 O(n) 时间复杂度，拒绝回溯攻击
- 只读规则在启动时编译一次，运行时复用编译后的 `*regexp.Regexp`

### 4.3 配置优先级

```
CLI 参数 > 配置文件（.skillguard.yaml）> 默认值
```

- `--ignore` 参数追加到配置文件的 ignore 列表
- 其他参数均以 CLI 参数为准覆盖配置文件

### 4.4 错误处理策略

- **非致命错误（文件读取失败、编码问题）：** 跳过该文件，继续扫描，退出码为 0
- **致命错误（配置/规则格式错误、目录不存在）：** 终止扫描，退出码为 2
- **正则匹配 panic 保护：** 自定义正则匹配在 `defer recover` 保护下执行

## 5. 接口设计

### 5.1 核心接口

```go
// 规则引擎接口
type Engine interface {
    Match(file *FileTarget) []*MatchResult
}

// 文件发现器接口
type FileDiscoverer interface {
    Discover(root string, opts *FileOpts) ([]*FileTarget, error)
}

// 输出渲染器接口
type Renderer interface {
    Render(report *ScanReport, opts *OutputOpts) (string, error)
}
```

### 5.2 CLI 接口

```
skill-guard [path...] [flags]
```

14 个参数：`--config`、`--rules`、`--severity`、`--json`、`--quiet`、`--verbose`、`--ignore`、`--ext-include`、`--ext-exclude`、`--max-size`、`--concurrency`、`--disable-builtin`、`--version`、`--help`

完整参数规范参见需求规格说明书第 4 章。

### 5.3 退出码

| 退出码 | 含义 |
|--------|------|
| 0 | 扫描完成，无风险 |
| 1 | 发现风险 |
| 2 | 运行错误 |

## 7. 性能设计

| 场景 | 目标 |
|------|------|
| 1000 文件扫描 | 比同类 Node.js 工具快 2 倍以上 |
| 10000 文件扫描 | 内存占用 < 50 MB |
| 单规则单文件 | 匹配耗时 < 1 ms |
| 1000 文件全规则 | 单规则耗时 < 50 ms |
| 大文件（10 MB） | 不 panic，线性能耗 |

### 7.1 性能优化措施

- 关键字预过滤减少正则调用
- 按行读取避免全文加载
- 编译时规则预编译
- Worker Pool 控制并发上限
- 文件大小限制防止 OOM

---

## 8. 安全性设计

- **不执行被扫描文件：** 只读分析，无执行路径
- **完全离线：** 无网络请求
- **Regex panic 保护：** defer/recover 包裹自定义规则匹配
- **文件编码检测：** 非 UTF-8 文件跳过，防止二进制文件注入
- **SELinux/AppArmor 兼容：** 无特殊权限要求
