## 1. 概述

skill-guard 是一款命令行安全扫描工具，采用**零外部存储依赖**的设计原则。所有数据以**文件形式**存储，无需数据库管理系统。本文档覆盖工具涉及的全部数据模型定义。

### 1.1 数据存储方式

| 数据类型 | 存储格式 | 存储位置 | 说明 |
|---------|---------|---------|------|
| 内置规则 | Go 源代码 + YAML | `rules/builtin_rules.yaml` | 编译时嵌入二进制 |
| 自定义规则 | YAML / JSON | 用户指定路径 | 运行时加载 |
| 配置文件 | YAML | `.skillguard.yaml` | 可选，位于工作目录 |
| 扫描报告 | JSON（运行时） | 标准输出 / 文件重定向 | `skill-guard --json > report.json` |
| 版本信息 | 编译时注入 | 二进制文件 | LDFLAGS 注入 |

## 2. 规则数据模型

### 2.1 规则结构体

```go
type Rule struct {
    ID          string   `yaml:"id" json:"id"`
    Name        string   `yaml:"name" json:"name"`
    Severity    string   `yaml:"severity" json:"severity"`
    Description string   `yaml:"description" json:"description"`
    Pattern     string   `yaml:"pattern" json:"pattern"`
    Keywords    []string `yaml:"keywords,omitempty" json:"keywords,omitempty"`
    FileTypes   []string `yaml:"file_types,omitempty" json:"file_types,omitempty"`
    Ref         string   `yaml:"ref,omitempty" json:"ref,omitempty"`
}
```

### 2.2 字段约束

| 字段 | 类型 | 必填 | 约束 |
|------|------|------|------|
| `id` | string | 是 | 全局唯一，格式 `SKL-\d{3}` 或 `USR-\w+`，长度 ≤ 32 |
| `name` | string | 是 | 长度 ≤ 100 |
| `severity` | string | 是 | 枚举值：`Critical` / `High` / `Medium` / `Low` |
| `description` | string | 是 | 长度 ≤ 500 |
| `pattern` | string | 是 | 合法 RE2 正则表达式，长度 ≤ 1000 |
| `keywords` | []string | 否 | 每个关键字长度 ≥ 2，最多 10 个 |
| `file_types` | []string | 否 | 每个以 `.` 开头，如 `.py`、`.sh` |
| `ref` | string | 否 | 合法 URL，长度 ≤ 500 |

### 2.3 规则文件格式（YAML）

```yaml
# rules/builtin_rules.yaml
rules:
  - id: "SKL-001"
    name: "硬编码 AWS Access Key"
    severity: "Critical"
    description: "检测硬编码的 AWS Access Key ID，防止云服务凭证泄露"
    pattern: "(?i)AKIA[A-Z0-9]{16}"
    keywords: ["AKIA"]
    ref: "https://docs.aws.amazon.com/IAM/latest/UserGuide/security-creds.html"

  - id: "SKL-002"
    name: "私钥文件泄露"
    severity: "Critical"
    description: "检测嵌入的 RSA/OPENSSH/EC 私钥内容"
    pattern: "-----BEGIN (RSA|OPENSSH|EC|DSA) PRIVATE KEY-----"
    keywords: ["BEGIN", "PRIVATE KEY"]
    ref: "https://www.ssh.com/academy/ssh/private-key"
```

### 2.4 规则文件格式（JSON）

```json
[
  {
    "id": "SKL-001",
    "name": "硬编码 AWS Access Key",
    "severity": "Critical",
    "description": "检测硬编码的 AWS Access Key ID",
    "pattern": "(?i)AKIA[A-Z0-9]{16}",
    "keywords": ["AKIA"],
    "ref": "https://docs.aws.amazon.com/IAM/latest/UserGuide/security-creds.html"
  }
]
```

### 2.5 规则严重级别枚举

```go
const (
    SeverityCritical = "Critical"
    SeverityHigh     = "High"
    SeverityMedium   = "Medium"
    SeverityLow      = "Low"
)

// 严重级别权重（用于排序和过滤）
var SeverityWeight = map[string]int{
    SeverityCritical: 4,
    SeverityHigh:     3,
    SeverityMedium:   2,
    SeverityLow:      1,
}

// 严重级别过滤阈值
func SeverityThreshold(level string) int {
    return SeverityWeight[level]
}
```

## 3. 配置文件数据模型

### 3.1 配置文件结构体

```go
type Config struct {
    Paths          []string `yaml:"paths,omitempty" json:"paths,omitempty"`
    Ignore         []string `yaml:"ignore,omitempty" json:"ignore,omitempty"`
    Rules          string   `yaml:"rules,omitempty" json:"rules,omitempty"`
    Severity       string   `yaml:"severity,omitempty" json:"severity,omitempty"`
    ExtInclude     []string `yaml:"ext_include,omitempty" json:"ext_include,omitempty"`
    ExtExclude     []string `yaml:"ext_exclude,omitempty" json:"ext_exclude,omitempty"`
    MaxSize        string   `yaml:"max_size,omitempty" json:"max_size,omitempty"`
    Concurrency    int      `yaml:"concurrency,omitempty" json:"concurrency,omitempty"`
    DisableBuiltin bool     `yaml:"disable_builtin,omitempty" json:"disable_builtin,omitempty"`
}
```

### 3.2 配置文件示例

```yaml
# .skillguard.yaml
ignore:
  - "tests/**"
  - "examples/**"
  - "*.log"

rules: "./my-custom-rules.yaml"
severity: "medium"

ext-include:
  - ".py"
  - ".sh"

ext-exclude:
  - ".md"
  - ".txt"

max-size: "5MB"
concurrency: 8
disable-builtin: false
```

### 3.3 默认值

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `paths` | `["."]` | 当前目录 |
| `ignore` | `[".git", "node_modules", "vendor", ...]` | 内置忽略列表 |
| `severity` | `""` | 不过滤 |
| `max-size` | `"10MB"` | 转换为 10 * 1024 * 1024 字节 |
| `concurrency` | `0` | 0 表示使用 `runtime.NumCPU()` |
| `disable-builtin` | `false` | 启用内置规则 |

## 4. 扫描结果数据模型

### 4.1 匹配结果

```go
type MatchResult struct {
    RuleID      string `json:"rule_id"`
    Severity    string `json:"severity"`
    FilePath    string `json:"file_path"`
    LineNumber  int    `json:"line_number"`    // 1-based
    LineContent string `json:"line_content"`   // 截取前 120 字符
    MatchType   string `json:"match_type"`     // "keyword" 或 "regex"
}
```

### 4.2 扫描报告

```go
type ScanReport struct {
    ScanTime    string         `json:"scan_time"`    // ISO 8601，如 "2026-04-29T10:30:00+08:00"
    Duration    string         `json:"duration"`     // 如 "1.234s"
    TotalFiles  int            `json:"total_files"`
    TotalIssues int            `json:"total_issues"`
    Results     []*MatchResult `json:"results"`
    Summary     *Summary       `json:"summary"`
}

type Summary struct {
    Critical int `json:"critical"`
    High     int `json:"high"`
    Medium   int `json:"medium"`
    Low      int `json:"low"`
}
```

## 11. 设计说明

### 11.1 为什么没有数据库

| 考量点 | 说明 |
|--------|------|
| **零依赖原则** | 引入数据库（即使是 SQLite）会增加部署复杂度，违背"零依赖单二进制"的核心承诺 |
| **无状态设计** | 每次扫描独立运行，不需要跨会话持久化数据 |
| **输出即存储** | `--json` 输出可重定向至文件，即为持久化的扫描记录 |
| **规则即代码** | 规则定义在 YAML/JSON 文件中，天然适合版本管理（Git） |

### 11.2 未来可扩展的存储需求

| 功能 | 可能的存储方式 |
|------|--------------|
| 规则远程更新 | HTTP 拉取规则文件，本地缓存 |
| 扫描历史追踪 | 可选 SQLite 嵌入（通过 build tag 控制） |
| 结果缓存 | 文件内容哈希 → 缓存扫描结果 |
| AI 辅助检测 | 本地 AI 模型文件 + 规则 Score 数据库 |
