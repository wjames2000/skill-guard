## 三、v1.1 详细执行计划

### Sprint 1: Scanner 测试 + 可靠性

```
Task 1: Scanner 单元测试（2h）
  - TestScan_FindsRisk: 扫描含 AKIA 的文件，验证输出含 SKL-001
  - TestScan_CleanDirectory: 扫描空目录，验证 TotalIssues=0
  - TestScan_EmptyDirectory: 扫描空目录，验证 Duration 非空
  - TestScan_WithCustomConfig: 传递自定义 Config，验证路径传递

Task 2: 正则 panic 保护（1h）
  - matcher.go 的 MatchLine 中增加 defer/recover
  - 自定义规则正则 panic 时跳过该规则，继续匹配

Task 3: 性能基准测试（2h）
  - internal/engine/bench_test.go: 单规则 1000 文件基准
  - internal/scanner/bench_test.go: 多规则并发基准
```

### Sprint 2: 兼容性 + 体验

```
Task 4: Windows 颜色兼容（1h）
  - 检测 os.Stdout 是否为终端
  - 非终端或 Windows 旧终端时 fallback 无颜色

Task 5: Symlink 跳过（0.5h）
  - discover.go 中 d.Type()&os.ModeSymlink 检查

Task 6: 进度条优化（1h）
  - scanner.go 中使用 fmt.Fprintf(os.Stderr, "\r进度: %d/%d") 单行刷新

Task 7: .gitignore 自动识别（2h）
  - 读取目标目录 .gitignore 文件
  - 解析忽略模式合并到 ignorePatterns
```
