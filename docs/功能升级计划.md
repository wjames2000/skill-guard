## 四、升级路线图

### v2.1 — 质量深化（2 周）

目标：提升测试覆盖率至 ≥80 用例，补齐遗漏的边缘场景。

| 任务 | 优先级 | 说明 | 工时 |
|------|--------|------|------|
| 新规则正面测试 | P0 | 为新增 68 条规则各写 2+ 正面测试 | 3d |
| 新规则负面测试 | P0 | 为新增规则各写 1-2 负面测试防误报 | 2d |
| Engine 补充测试 | P1 | 大文件、并发、空规则、二进制文件 | 0.5d |
| File 补充测试 | P1 | symlink、超大文件、非 UTF-8 目录 | 0.5d |
| Output 补充测试 | P1 | SeverityFilter 边界、空报告 | 0.5d |
| Report 补充测试 | P1 | 空结果、全部同级别排序 | 0.5d |
| Config 补充测试 | P1 | 文件损坏、无权限、编码问题 | 0.5d |
| Stress 测试 | P2 | 10000 文件内存基准、大文件性能基准 | 1d |

### v2.2 — 规则生态（2 周）

目标：建立社区规则贡献机制，完善规则生命周期管理。

| 任务 | 优先级 | 说明 | 工时 |
|------|--------|------|------|
| 规则版本管理 | P1 | `update` 命令支持规则版本对比，按需回滚 | 1d |
| 规则贡献模板 | P1 | `skill-guard new-rule` 交互式规则创建向导 | 1d |
| 规则评分系统 | P2 | 规则热度、准确率数据上报（opt-in） | 1.5d |
| 规则自动测试 | P2 | 提交规则时自动运行正面/负面测试套件 | 1d |
| 规则导出/分享 | P1 | `skill-guard rules export` 导出自定义规则 | 0.5d |
| 社区规则源 | P2 | 从 GitHub 社区仓库拉取规则 | 1d |

### v2.3 — 平台集成（3 周）

目标：进入开发者工具链，覆盖更多使用场景。

| 任务 | 优先级 | 说明 | 工时 |
|------|--------|------|------|
| GitHub Action | P1 | 官方 Action，CI 一键集成 | 2d |
| Pre-commit 生态 | P1 | pre-commit hooks 官方仓库收录 | 1d |
| VS Code 插件 | P2 | 编辑器内实时扫描 + 问题面板 | 5d |
| JetBrains 插件 | P2 | IntelliJ Platform 插件 | 5d |
| LSP 协议支持 | P2 | Language Server Protocol，多编辑器通用 | 3d |
| Homebrew 官方仓库 | P1 | 提交至 Homebrew 官方 core 仓库 | 0.5d |

### v2.4 — 高级引擎（3 周）

目标：引入脚本化规则和智能化分析能力。

| 任务 | 优先级 | 说明 | 工时 |
|------|--------|------|------|
| LUA 脚本规则引擎 | P1 | 嵌入 Go LUA VM，支持复杂逻辑规则 | 3d |
| YARA 规则兼容 | P1 | 支持 YARA 规则格式加载 | 2d |
| AI 检测增强 | P2 | 批量 AI 分析、缓存、置信度阈值配置 | 2d |
| AI 检测本地化 | P2 | 集成 Ollama 自动下载推荐模型 | 2d |
| 语义分析引擎 | P2 | AST 解析 + 模式匹配，降低正则误报 | 3d |

### v3.0 — 平台化（远期）

| 任务 | 说明 | 复杂度 |
|------|------|--------|

| 团队管理 | 团队共享配置、规则、扫描报告 | 高 |
| 可视化报告 | HTML 报告 + 趋势图表 | 中 |
| 定时扫描 | cron/调度器集成，定期扫描 | 低 |
