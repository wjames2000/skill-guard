# 规则贡献指南

## 如何贡献规则

### 前提条件

- 熟悉 Go 正则语法（RE2）
- 了解至少一种编程语言的安全最佳实践

### 规则格式

每条规则为 YAML 格式，包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 唯一标识，自定义规则用 `USR-` 前缀 |
| `name` | string | 是 | 短名称，如「检测硬编码密码」 |
| `severity` | string | 是 | 严重级别: Critical / High / Medium / Low |
| `description` | string | 是 | 规则的详细说明 |
| `pattern` | string | 是 | RE2 兼容的正则表达式 |
| `keywords` | []string | 否 | 预过滤关键字，提高扫描性能 |
| `file_types` | []string | 否 | 限定扫描的文件扩展名 |
| `ref` | string | 否 | 参考资料链接 |

### 快速开始

```bash
# 生成规则模板
skill-guard rules new

# 编辑模板
vim custom-rule-20260429-120000.yaml

# 测试规则
skill-guard rules test custom-rule-20260429-120000.yaml

# 使用规则扫描
skill-guard ./skills --rules custom-rule-20260429-120000.yaml
```

### 编写原则

1. **关键词预过滤**: 设置精确的 `keywords` 避免对无关文件执行正则
2. **文件类型限定**: 使用 `file_types` 限制规则适用的文件类型
3. **避免误报**: 确保正则在安全代码上不会误触发
4. **性能优先**: 正则复杂度保持 O(n)，避免回溯

### 提交规则

1. Fork 仓库
2. 将规则文件放入 `rules/community/` 目录
3. 在 `rules/community-index.yaml` 中添加条目
4. 提交 Pull Request
