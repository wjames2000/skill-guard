## 1. 产品概述

### 1.1 产品定位

`skill-guard` 是一款用 Go 语言编写的命令行安全扫描工具，专注于检测 AI 辅助开发中第三方技能（Skill）文件的安全风险。以零依赖的单一可执行文件形态交付，提供快速、可扩展、可自定义规则的安全扫描能力。

### 1.2 产品名与代号

- **产品名：** skill-guard
- **命令行调用名：** `skill-guard`
- **包名（Go）：** `github.com/wjames2000/skill-guard`

### 1.3 核心价值

- **零依赖部署：** 单二进制文件，无需 Node.js、Python 等运行时环境
- **即开即用：** 内置 30+ 安全检测规则，覆盖常见风险模式
- **CI/CD 友好：** 结构化 JSON 输出、语义化退出码
- **可扩展：** 自定义规则、接口化引擎设计

### 1.4 目标用户

| 用户角色 | 使用场景 | 核心诉求 |
|----------|---------|---------|
| **开发者** | 集成第三方技能前的本地审查 | 快速扫描、清晰的终端输出 |
| **团队/企业** | CI/CD 流水线中的安全卡点 | 结构化输出、退出码判断、配置管理 |
| **安全工程师** | 深度审查与规则定制 | 自定义规则、细粒度扫描控制 |

### 2.2 模块职责

| 模块 | 职责 | 关键接口 |
|------|------|---------|
| **CLI 入口** | 解析命令行参数、调用扫描流程、处理退出码 | `Execute() error` |
| **配置管理** | 读取 `.skillguard.yaml`、合并 CLI 参数、构建 Config 对象 | `LoadConfig(path string) *Config` |
| **文件发现** | 递归遍历目录、应用 ignore 规则、构建文件列表 | `DiscoverFiles(root string, opts *FileOpts) []*FileTarget` |

| **结果聚合** | 合并匹配结果、按严重级别排序、去重 | `Aggregate(results []*MatchResult) *Report` |
| **输出渲染** | 根据输出模式渲染结果 | `Render(report *Report, opts *OutputOpts) (string, error)` |

## 3. 功能需求

### 3.1 FR-1 目录扫描

**描述：** 支持对本地文件系统的目录进行递归安全扫描。

**详细需求：**

| 需求 ID | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-1.1 | 支持扫描单个目录路径，如 `skill-guard ./skills` | P0 |
| FR-1.2 | 支持同时扫描多个目录路径，如 `skill-guard ./skills ./community` | P0 |
| FR-1.3 | 递归遍历子目录，深度不限（由文件系统限制） | P0 |
| FR-1.4 | 默认忽略 `.git/`、`node_modules/`、`vendor/`、`__pycache__/`、`.svn/` 等目录 | P0 |
| FR-1.5 | 支持通过 CLI 参数 `--ignore` 或 `-i` 添加额外忽略路径或 glob 模式 | P1 |
| FR-1.6 | 支持通过配置文件配置 ignore 列表 | P1 |
| FR-1.7 | 扫描时跳过符号链接（Symlink）以规避循环引用风险 | P2 |

**忽略优先级：** CLI 参数 > 配置文件 > 默认忽略列表（后者的忽略规则在前者未指定时才生效）。

### 3.2 FR-2 文件类型支持

**描述：** 确定哪些文件类型会被纳入扫描范围。

**详细需求：**

| 需求 ID | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-2.1 | 默认扫描扩展名：`.md`、`.json`、`.yaml`、`.yml`、`.py`、`.sh`、`.js`、`.ts`、`.toml`、`.xml`、`.txt`、`.cfg`、`.conf`、`.ini`、`.env`、`.bat`、`.ps1`、`.rb`、`.php`、`.lua` | P0 |
| FR-2.2 | 支持 `--ext-include` 参数限制只扫描特定扩展名（如 `--ext-include .py,.sh`） | P1 |
| FR-2.3 | 支持 `--ext-exclude` 参数排除特定扩展名（如 `--ext-exclude .md,.txt`） | P1 |
| FR-2.4 | 默认跳过二进制文件（通过检测前 512 字节是否为有效 UTF-8 文本） | P0 |
| FR-2.5 | 文件大小上限默认 10 MB（可配置 `--max-size`），超限文件跳过并警告 | P1 |
| FR-2.6 | 非 UTF-8 编码文件跳过并给出警告，不中断扫描 | P1 |

**优先级规则：** `--ext-include` 指定后，仅扫描指定扩展名（`--ext-exclude` 在其基础上进一步排除）。两者均未指定时，使用默认扩展名列表。

### 3.3 FR-3 规则检测引擎

**描述：** 核心检测能力，内置 30+ 条安全规则覆盖常见风险模式。

**详细需求：**

| 需求 ID | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-3.1 | 内置规则不少于 30 条，覆盖五大安全类别 | P0 |
| FR-3.2 | 规则分四个严重级别：Critical、High、Medium、Low | P0 |
| FR-3.3 | 支持正则表达式匹配（使用 Go RE2 语法） | P0 |
| FR-3.4 | 支持关键字预过滤：仅当文件包含指定关键字时才执行正则匹配，提升大文件扫描性能 | P0 |
| FR-3.5 | 支持按文件类型（扩展名）限定规则的适用范围 | P1 |
| FR-3.6 | 支持按行匹配和全文匹配两种模式 | P1 |

**五大安全类别：**

| 类别 | 描述 | 示例规则数量 |
|------|------|-------------|
| **密钥泄露** | 硬编码 API Key、Token、私钥、数据库连接串 | ≥ 6 |
| **命令执行** | exec、system、eval、反引号执行等 | ≥ 6 |
| **恶意文件操作** | 递归删除、任意文件写入、文件权限篡改 | ≥ 5 |
| **网络请求滥用** | curl/wget 到外部地址、下载执行、反向 Shell | ≥ 5 |
| **信息窃取与混淆** | 环境变量泄露、敏感文件读取、Base64/Hex 混淆载荷 | ≥ 5 |

**匹配优先级：** 同一个文件行可能匹配多条规则，所有匹配结果均保留。同一条规则在同一文件中的多次匹配结果合并为一条（记录首次匹配行号）。

### 3.4 FR-4 自定义规则

**描述：** 用户可加载外部规则文件扩展或覆盖内置规则。

**详细需求：**

| 需求 ID | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-4.1 | 支持通过 `--rules` 参数指定外部规则文件（YAML 或 JSON 格式） | P0 |
| FR-4.2 | 自定义规则支持追加到内置规则之后，或通过相同 `id` 覆盖内置规则 | P0 |
| FR-4.3 | 自定义规则的格式与内置规则保持一致 | P0 |
| FR-4.4 | 支持配置文件中指定自定义规则文件路径 | P1 |
| FR-4.5 | 支持 `--disable-builtin` 参数禁用所有内置规则（仅使用自定义规则） | P1 |

**规则合并策略：**

```
1. 加载内置规则列表
2. 加载自定义规则文件
3. 对每条自定义规则：
   a. 如果 id 与内置规则重复 → 覆盖内置规则
   b. 如果 id 不重复 → 追加到列表末尾
4. 如果指定了 --disable-builtin → 仅使用自定义规则
```

### 3.5 FR-5 输出格式

**描述：** 支持多种输出模式，满足不同场景需求。

**详细需求：**

| 需求 ID | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-5.1 | 默认输出：终端彩色显示，展示文件路径、行号、严重级别、规则 ID、匹配行内容摘要 | P0 |
| FR-5.2 | `--json` 参数：输出完整 JSON 格式，包含元信息（扫描时间、扫描文件数、总耗时） | P0 |
| FR-5.3 | `--quiet` 参数：仅显示存在问题的文件列表，不显示扫描进度、总结信息 | P1 |
| FR-5.4 | `--severity` 参数：按严重级别过滤输出，如 `--severity high` 仅显示 High 及以上 | P1 |
| FR-5.5 | 输出结果按严重级别排序：Critical → High → Medium → Low | P0 |
| FR-5.6 | 每个匹配结果输出行号时使用 1-based 行号 | P0 |

### 3.6 FR-6 退出码

**描述：** 通过退出码传递扫描结果状态，便于 CI 集成。

| 退出码 | 含义 | 触发条件 |
|--------|------|---------|
| `0` | 扫描完成，无风险 | 扫描正常结束，未发现任何匹配项 |
| `1` | 发现风险 | 扫描正常结束，至少有一个匹配项 |
| `2` | 运行错误 | 指定的目录不存在、无读取权限、配置文件语法错误等 |

**与 `--severity` 的关系：** 当指定 `--severity high` 时，退出码 `1` 仅在发现 High 及以上风险时触发。如果仅发现 Medium/Low，退出码为 `0`。

### 3.7 FR-7 配置管理

**描述：** 通过配置文件管理常用扫描选项，减少命令行参数输入。

**详细需求：**

| 需求 ID | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-7.1 | 默认在当前工作目录查找 `.skillguard.yaml` 配置文件 | P0 |
| FR-7.2 | 支持通过 `--config` 参数指定配置文件的路径 | P0 |
| FR-7.3 | CLI 参数优先级高于配置文件 | P0 |
| FR-7.4 | 配置文件支持以下字段：ignore 路径、自定义规则路径、默认严重级别过滤、扩展名包含/排除 | P1 |
| FR-7.5 | 配置文件不存在时不报错（非必选） | P0 |

### 3.8 FR-8 扫描性能

**描述：** 利用 Go 并发实现高性能文件扫描。

| 需求 ID | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-8.1 | 文件扫描并发执行，goroutine 数量默认为 CPU 核数 | P0 |
| FR-8.2 | `--verbose` 参数显示实时扫描进度（已扫描文件数 / 总文件数） | P1 |
| FR-8.3 | `--concurrency` 参数控制并发数（如 `--concurrency 4`） | P2 |
| FR-8.4 | 1000 个文件扫描场景下性能不低于同类 Node.js 工具的 2 倍 | P0 |

## 6. 规则引擎规范

### 6.1 规则数据结构

```go
type Rule struct {
    ID          string   `yaml:"id" json:"id"`                      // 唯一标识，如 "SKL-001"
    Name        string   `yaml:"name" json:"name"`                  // 短名称
    Severity    string   `yaml:"severity" json:"severity"`          // Critical / High / Medium / Low
    Description string   `yaml:"description" json:"description"`    // 规则目的说明
    Pattern     string   `yaml:"pattern" json:"pattern"`            // RE2 正则表达式
    Keywords    []string `yaml:"keywords,omitempty" json:"keywords,omitempty"`       // 预过滤关键字
    FileTypes   []string `yaml:"file_types,omitempty" json:"file_types,omitempty"`   // 限定文件扩展名
    Ref         string   `yaml:"ref,omitempty" json:"ref,omitempty"`                 // 参考链接
}
```

### 6.2 匹配结果数据结构

```go
type MatchResult struct {
    RuleID      string `json:"rule_id"`              // 匹配的规则 ID
    Severity    string `json:"severity"`             // 严重级别
    FilePath    string `json:"file_path"`            // 文件路径
    LineNumber  int    `json:"line_number"`          // 行号（1-based）
    LineContent string `json:"line_content"`         // 匹配行内容摘要
    MatchType   string `json:"match_type,omitempty"` // "keyword" 或 "regex"
}
```

### 6.3 扫描报告数据结构

```go
type ScanReport struct {
    ScanTime    string         `json:"scan_time"`          // 扫描时间 ISO 8601
    Duration    string         `json:"duration"`           // 扫描耗时
    TotalFiles  int            `json:"total_files"`        // 总扫描文件数
    TotalIssues int            `json:"total_issues"`       // 总发现风险数
    Results     []*MatchResult `json:"results"`            // 匹配结果列表
    Summary     *Summary       `json:"summary"`            // 按严重级别汇总
}

type Summary struct {
    Critical int `json:"critical"`
    High     int `json:"high"`
    Medium   int `json:"medium"`
    Low      int `json:"low"`
}
```

### 6.4 匹配执行流程

```
对于每个文件:
  1. 读取文件（按行读取，支持大文件）
  2. 检查编码（非 UTF-8 跳过）
  3. 检查文件大小（超限跳过）
  4. 遍历所有启用的规则:
     a. 如果规则有 FileTypes 限定 → 检查文件扩展名是否匹配
     b. 如果规则有 Keywords → 检查文件是否包含任一关键字（快速字符串查找）
     c. 如果关键字预过滤通过 → 对每一行执行正则匹配
     d. 匹配成功 → 记录 MatchResult
  5. 当前文件处理完成 → 返回结果列表

多文件并发: 每个文件在一个独立的 goroutine 中处理，通过 channel 收集结果
```

## 9. 错误处理

### 9.1 错误分类与处理策略

| 错误类型 | 处理策略 | 退出码 | 示例 |
|---------|---------|--------|------|
| 目录不存在 | 输出错误，终止扫描 | 2 | `skill-guard /nonexistent` |
| 无读取权限 | 跳过该文件/目录，继续 | 0 | 某个子目录无读权限 |
| 非 UTF-8 文件 | 跳过并警告 | 0 | 二进制文件 |
| 文件超限 | 跳过并警告 | 0 | 超过 `--max-size` 的文件 |
| 配置文件格式错误 | 输出错误，终止扫描 | 2 | YAML 语法错误 |
| 自定义规则格式错误 | 输出错误，终止扫描 | 2 | 缺少必填字段 |
| 自定义规则正则非法 | 输出错误，终止扫描 | 2 | 无效 RE2 语法 |
| 路径为文件而非目录 | 直接扫描该文件 | 0 | `skill-guard somefile.py` |

ERROR: <错误描述>

### 10.3 版本号规范

遵循语义化版本号（SemVer）：

```
v<major>.<minor>.<patch>
```

- major：不兼容的 CLI 或规则格式变更
- minor：新增功能或规则，向后兼容
- patch：Bug 修复，向后兼容

| Skill | 供 AI 辅助开发工具使用的技能文件，以 Markdown、JSON、YAML 等形式存在 |
